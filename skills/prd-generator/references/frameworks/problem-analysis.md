# Problem Analysis Framework

## Purpose
Guide systematic analysis of problems and opportunities for PRD development.

## Framework Steps

### 1. Define the Problem
- **What is the specific pain point?**
  - Be concrete and specific
  - Use real user quotes or data when possible
  - Avoid solution-oriented language
  
- **Who experiences this problem?**
  - Primary affected users
  - Secondary stakeholders
  - Scale/magnitude of impact

- **When does this problem occur?**
  - Specific workflows or scenarios
  - Frequency of occurrence
  - Context and triggers

### 2. Validate Problem Significance
- **Evidence of the problem:**
  - User research data
  - Support tickets or feedback
  - Usage analytics
  - Market research

- **Impact assessment:**
  - User productivity/efficiency loss
  - Business revenue impact
  - Competitive disadvantage
  - Strategic importance

### 3. Root Cause Analysis
- Use "5 Whys" technique
- Distinguish symptoms from root causes
- Identify underlying systemic issues

### 4. Define Goals (2-3 Maximum)
Goals should be:
- **Specific:** Clear and unambiguous
- **Measurable:** Can track progress
- **Achievable:** Within scope and resources
- **Relevant:** Aligned with business strategy
- **Time-bound:** Clear timeline
