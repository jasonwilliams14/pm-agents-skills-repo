# PRD Outlines

Select the outline that best fits the nature of the feature or enhancement.

## 1. Standard PRD
Best for general product features.
1. Executive Summary
2. Problem Statement & Goals
3. Target Audience
4. User Stories
5. Functional Requirements
6. Non-Functional Requirements
7. Analytics & Success Metrics
8. Go-to-Market (GTM) Plan

## 2. Technical PRD
Best for infrastructure, backend, or API changes.
1. Overview & Objective
2. System Architecture Impact
3. API Specifications / Schema Changes
4. Performance & Scalability Requirements
5. Security & Privacy Considerations
6. Migration & Rollback Strategy
7. Testing Strategy

## 3. UX-Focused PRD
Best for frontend, design-heavy, or user-interaction features.
1. Design Vision & User Experience Goals
2. User Personas & Scenarios
3. User Flow Diagrams
4. UI Requirements & Key Screens
5. Accessibility Requirements
6. Interaction Details
7. Design Validation & User Testing
