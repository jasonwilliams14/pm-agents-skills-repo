# PM Standards

## Overview
This skill enforces the "Jason Standard" for Product Management, focusing on high-signal output, empirical validation through prototypes (POC-First), and rigorous prioritization.

## Core Philosophy: POC-First
Every proposal must conclude with a Value Proposition and Competitive Differentiation. Documentation should ideally be born from functional prototypes or clear technical feasibility studies.

## Core Capabilities

### 1. Discovery & Needs Analysis (JTBD)
Use the Jobs-to-be-Done framework to define user needs.
- Reference: [jtbd.md](references/jtbd.md)

### 2. Prioritization (RICE)
Use RICE scoring for all feature requests and backlog items.
- Reference: [rice.md](references/rice.md)

### 3. Success Metrics & Observability (OTEL)
No feature is complete without a Success Metrics section that includes OpenTelemetry (OTEL) implementation details.
- Reference: [metrics.md](references/metrics.md)

### 4. Stakeholder-Aligned Communication
Always provide three versions of summaries (Executive, Engineering, Product).
- Reference: [templates.md](references/templates.md)

## Usage Guidelines
- When asked to draft a PRD, follow the structure in [templates.md](references/templates.md).
- When asked to prioritize, use the framework in [rice.md](references/rice.md).
- When asked to summarize a project or feature, use the Stakeholder Lens from [templates.md](references/templates.md).
